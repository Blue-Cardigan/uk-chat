# ChatGB Privacy Policy (Draft)

Last updated: 2026-03-30

## 1. Controller

ChatGB is the data controller for personal data processed through the product.

Contact: privacy@explorethekingdom.co.uk

## 2. What we collect

- Account data: email address and profile identifiers.
- Product usage data: chat conversations, uploaded document extracts, council-mode requests, model usage counters.
- Access/admin data: invite records, token onboarding state, and audit logs for admin actions.

## 3. Why we process data (lawful bases)

- Account access and authentication: performance of a contract (UK GDPR Article 6(1)(b)).
- Product operation (chat responses, council mode, usage limits): legitimate interests (Article 6(1)(f)).
- Optional personalisation/synthetic profile features: consent (Article 6(1)(a)); explicit consent where special category data is involved (Article 9(2)(a)).

## 4. Processors and third parties

ChatGB uses the following processors/integrations:

- Supabase (authentication and database hosting)
- OpenRouter (LLM inference)
- Resend (transactional email)
- MCP infrastructure (tool access and token issuance)

Data Processing Agreements (DPAs) must be maintained for each processor.

## 5. Sharing links

If a user enables public sharing for a conversation, anyone with the share link can access that conversation until sharing is revoked or the link expires.

## 6. Retention

- Default retention target for chat and council records: 12 months.
- Audit logs are retained for security/accountability.
- Users can request erasure and can delete their own account in-product.

## 7. User rights

Users can:

- Access and export their data.
- Correct profile information.
- Delete account and associated data (right to erasure).
- Object to processing based on legitimate interests.

## 8. International transfers

Some processors may process data outside the UK. Appropriate safeguards must be in place (for example, UK IDTA/SCCs where required).

## 9. Security

Controls include access controls, row-level security, audit logging, and encryption for sensitive token material.

## 10. Changes

Material changes to this policy will be versioned and surfaced in-product.
